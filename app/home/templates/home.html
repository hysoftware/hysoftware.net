Process

First step: test

Estimating the delivery date on software industries is difficult.
However, it is able to be estimated by spliting the software by modules, functionalities, tasks, etc...
After split, It needs to code a software to check whether the function works or not.
In addition, this process can be handled before coding actual apps as far as possible.
This is one of the feature of Agile Development and it is called TDD or BDD.
By doing above, the written tests ensure that the code works even if the code is heavily-changed.

Second step: code

This is the most exciting part of coding.
Coders are feeling all-seeing and all-knowing while they are coding.
After the coding is finished, the goal of task is almost there!

Final step: refactor

The finished code is dirty, un-maintainable, but it just works.
This is why the code is written for passing tests, not for humans.
The code should be written for humans, therefore re-styling the code is needed.
This is just re-styling the code, and the behavior is not changed and it is guranteed by unit tests.

After the refactoring, the task is finished!

By repeating above steps, Hiro can code what your/your company's want.
